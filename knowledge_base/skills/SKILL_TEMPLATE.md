# Skill 模板 - 高质量标准

> 此模板用于创建新的 Skill，确保结构统一、信息密度高、可被 Agent 正确理解和执行。

---

## 模板说明

### YAML Frontmatter（必需）

```yaml
---
name: [英文名称，用于程序识别]
display_name: [中文显示名称]
level: [foundation | general | domain | complex]
domain: [所属领域，如 marketing, education, manufacturing]
version: v1.0
triggers:
  - [触发关键词1]
  - [触发关键词2]
dependencies:
  - [依赖的其他 Skill，可选]
tools:
  - [需要的工具，如 python_run, read_file]
---
```

### 核心原则

1. **高信息密度**：每句话都有价值，避免冗余
2. **可执行性**：步骤清晰，可直接执行
3. **可验证性**：每个步骤有明确的验证标准
4. **上下文友好**：便于 LLM 理解和遵循

## 信息密度优化指南

### 好的写法（高密度）

```markdown
### Step 1: 文件解析

**目标**：提取招标文件结构和关键信息

**方法**：
1. `read_file` 读取文件
2. 识别章节：资格要求、技术规格、评分标准、商务条款
3. 提取表格数据

**验证**：章节完整性 ≥ 95%

**输出**：结构化的章节列表和关键信息
```

### 差的写法（低密度）

```markdown
### 第一步：解析招标文件

首先，我们需要读取招标文件。可以使用 read_file 工具来读取文件内容。
读取完成后，需要识别文件中的各个章节，包括资格要求章节、技术规格章节、
评分标准章节和商务条款章节等。同时还需要提取文件中的表格数据。
最后需要验证提取的信息是否完整。
```

### 区别

| 方面 | 高密度 | 低密度 |
|------|-------|-------|
| Token 数 | ~50 | ~120 |
| 信息量 | 相同 | 相同 |
| 可执行性 | 高 | 低 |
| LLM 理解 | 容易 | 需要解析 |
