# Debug 方法论

## 适用场景
- 程序报错、行为异常、回归问题定位。
- 需要收集日志与运行时证据的故障排查。

## 执行步骤
1. 收集现象：错误信息、复现步骤、预期 vs 实际。
2. 生成假设：至少 2 个候选根因与验证方案。
3. 验证假设：一次只验证一个变量，记录证据链。
4. 最小修复：仅修改必要代码，保持可回滚。
5. 回归验证：原问题 + 邻近场景，清理临时调试代码。

## 质量门
- 结论必须是“现象→证据→定位→修复”的闭环。
- 无复现/无证据时不得输出确定性根因。
- 修复后必须给预防措施与复发监控建议。

## 输出要求
- 根因报告、修复说明、验证结果、残余风险。
- 信息不足时输出 `missing_information` 与补采集动作。
