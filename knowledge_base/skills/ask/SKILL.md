# Ask 方法论

## 适用场景
- 理解代码、架构、流程与设计取舍。
- 回答技术问题并给可执行建议。
- 需要“先分析不改代码”的咨询任务。

## 执行步骤
1. 明确问题边界与输出格式，不清楚先澄清。
2. 只读收集证据（`read_file`/`grep`/`search_knowledge`）。
3. 区分事实与判断，按技术/业务/风险三维分析。
4. 给推荐方案与注意事项，附来源路径。

## 质量门
- 关键结论必须有来源，不允许无依据强断言。
- 必须包含下一步建议（可执行动作）。
- 有不确定性时显式标注，并给验证路径。

## 禁止项
- 不修改文件，不执行高副作用命令。
- 若需改代码，明确建议切换 Agent 模式。
