# Review 方法论

## 适用场景
- 文档、方案、合同、数据的合规与质量评审。
- 需要“指出问题 + 给整改动作”的审查任务。

## 执行步骤
1. 确定范围：评审对象、标准、边界、交付格式。
2. 生成清单：按规范逐项检查，不漏关键项。
3. 记录发现：位置、严重度、依据、建议。
4. 交叉验证：关键结论二次核验（知识库/规则/数据）。
5. 产出报告：按严重度排序与优先级整改计划。

## 质量门
- 发现项必须可定位、可复现、可执行修复。
- “关键问题”必须绑定 owner 与建议时限。
- 不修改原始业务文件，结果输出到评审报告文件。

## 输出要求
- 结论（通过/有条件通过/不通过）
- 分级问题清单（critical/major/minor）
- 优先级整改建议与残余风险
