# 行业解决方案模板框架

技术方案撰写时可按行业选用以下框架，并与评标办法中的技术分项一一对应。

## 通用结构（与多数评标技术分项对齐）

1. **项目理解与需求分析**：招标需求摘要、关键指标、约束条件。
2. **总体方案/架构设计**：系统架构或实施总体思路、技术路线。
3. **详细设计与实现**：功能/模块设计、关键技术、接口与集成。
4. **实施计划**：阶段划分、里程碑、交付物、时间表。
5. **质量与保障措施**：质量保证、测试、验收、风险控制。
6. **团队与服务**：项目组织、人员配置、培训、售后服务。

## IT 类项目（软件开发、系统集成、云服务等）

- 需求理解 → 总体架构（业务架构/技术架构/部署架构）→ 功能与模块设计 → 开发与测试计划 → 部署与运维 → 团队与售后。
- 评分项常含：方案完整性、技术先进性、实施可行性、团队能力、服务保障。用语须与评标办法一致。

## 建筑/工程类项目

- 工程理解 → 施工组织设计 → 关键工序与工艺 → 进度计划与资源配置 → 质量安全文明措施 → 组织架构与人员。
- 评分项常含：施工组织、关键技术、工期保证、质量安全、人员与设备。

## 咨询/服务类项目

- 对需求的理解 → 工作思路与方法论 → 工作内容与成果 → 实施计划与团队 → 质量保证与创新点。
- 评分项常含：理解深度、方法科学性、成果可交付性、团队经验、创新。

## 使用建议

- 先提取招标中技术评分项及子项，再按上述框架映射到章节；每章对应哪些评分点应在方案中明确标注。
- 检索：`search_knowledge("技术方案模板 方案架构 行业方案")`；详细模板见 domain/procurement/references/templates/。
