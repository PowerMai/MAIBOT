# 项目类型分类

**领域**: 招投标 / 采购  
**用途**: 供 Agent 与 search_knowledge 检索使用

---

## 1. 按采购标的分类

| 类型 | 说明 | 典型评分侧重 |
|------|------|----------------|
| **IT/信息化** | 软件、系统集成、云服务、运维等 | 技术方案、案例、团队、实施与保障 |
| **咨询** | 规划、设计、监理、评估等 | 方法论、团队资质、类似项目经验 |
| **工程** | 施工、设备安装、装修等 | 施工组织、安全、工期、资质与业绩 |
| **货物/设备采购** | 硬件、设备、产品采购 | 产品规格、供货与售后、价格 |

---

## 2. 按采购方式（常见）

| 类型 | 说明 |
|------|------|
| 公开招标 | 公开邀请不特定投标人，程序与时限按法规执行 |
| 邀请招标 | 向特定投标人发出投标邀请，适用于法定允许情形 |
| 竞争性谈判 | 与多家供应商谈判，可多轮报价与方案优化 |
| 竞争性磋商 | 类似竞争性谈判，强调磋商与综合评审 |
| 单一来源 | 向唯一供应商采购，须符合法定条件 |

---

## 3. 分析时的类型识别

在解析招标文件时，可通过以下关键词初步判断项目类型（详见 `01_ANALYZE_BIDDING_DOCUMENT_V2.md` 的 INFO_EXTRACTION_MATRIX）：

- **项目/采购/招标** → 项目名称与编号  
- **IT/咨询/工程/采购** → 项目类型枚举  
- **预算、总价、采购金额** → 金额与规模  
- **需求、功能、性能指标** → 功能/技术需求  
- **评分、权重、技术分/商务分/价格分** → 评分结构  

项目类型影响投标文件结构、篇幅分配与评分策略；高权重技术分的项目应在技术方案章节投入更多篇幅与证据。
