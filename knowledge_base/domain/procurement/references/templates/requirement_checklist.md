# 需求检查清单

编制响应矩阵与标书、合规检查时，可按下列类别逐条核对招标要求，确保不遗漏、逐条响应。

---

## 资格条款

- [ ] 企业资质（等级、行业、有效期）
- [ ] 业绩要求（类似项目数量、金额、时间范围）
- [ ] 人员要求（项目经理、技术负责人、资格证）
- [ ] 财务与信用（注册资本、审计报告、无重大违法）
- [ ] 其他资格（联合体、分包、授权等）

每项须在投标文件中标明对应条款位置与证明材料。

---

## 技术条款

- [ ] 技术方案与实施方案（架构、功能、交付物）
- [ ] 技术参数与指标（性能、兼容性、标准）
- [ ] 人员配置与组织（项目组名单、职责、投入）
- [ ] 质量与验收（质量标准、验收方式、保修）
- [ ] 工期与里程碑（开工、关键节点、竣工）

技术方案用语应与评标办法中的评分项一致，便于评委对应打分。

---

## 商务条款

- [ ] 报价范围与口径（含税、含运、含培训等）
- [ ] 付款方式与节点
- [ ] 履约担保与保证金
- [ ] 违约责任与赔偿
- [ ] 保修期与售后服务

报价清单须与招标子目或价格分计算口径一致。

---

## 格式与密封

- [ ] 投标文件组成与顺序（按「投标文件格式」）
- [ ] 封面、目录、签章、骑缝章
- [ ] 正副本数量与标识
- [ ] 密封方式与包封标识
- [ ] 递交时间与地点

形式废标多与密封、时间、份数、签章有关，递交前应逐项核对。

---

## 实质性条款清单

- [ ] 招标文件中带*或明确为「实质性」的条款已全部列出
- [ ] 响应矩阵中每条实质性条款均有响应方式与证明材料
- [ ] 负偏离（如有）已填写偏离表并说明理由；正偏离已声明

未响应实质性条款将导致废标，须 100% 覆盖并注明条款位置。

---

## 使用建议

- 招标分析阶段：用本清单提取招标要求，生成三张表（废标条款清单、评分权重表、陷阱条款标注）。
- 标书撰写阶段：用本清单核对响应矩阵与各章节，确保逐条对应。
- 合规检查阶段：用本清单做形式与实质符合性检查。

执行指引与输出规范见 `knowledge_base/skills/marketing/bidding/BUNDLE.md`。
