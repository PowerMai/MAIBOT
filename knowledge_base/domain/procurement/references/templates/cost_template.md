# 成本估算模板

用于投标报价前的成本匡算与价格分测算，便于在预算范围内合理定价。

---

## 1. 成本构成（示例）

| 类别 | 子项 | 说明 | 金额（元） |
|------|------|------|------------|
| 人力 | 项目团队工资/人天 | 人天 × 单价 × 工期 | |
| 人力 | 管理/协调 | 项目经理、PMO 等 | |
| 设备/软件 | 硬件/ License | 按方案清单 | |
| 设备/软件 | 第三方服务 | 外包、云服务等 | |
| 其他 | 差旅、培训、验收 | 按招标要求 | |
| 其他 | 税费、不可预见费 | 按比例或固定 | |
| **合计** | **成本小计** | | |
| | 利润率（%） | 目标毛利 | |
| **报价** | **投标总价** | 成本 × (1 + 利润率) 或 分项汇总 | |

---

## 2. 价格分测算（与评分规则配合）

当招标采用「最低价得满分」或「基准价法」时，可先匡算成本与保本价，再结合竞争情况设定目标报价：

| 项目 | 数值 |
|------|------|
| 成本合计（元） | |
| 保本价（元） | 成本 × (1 + 最低目标利润率) |
| 招标控制价/预算（元） | 来自招标文件 |
| 目标报价（元） | 在保本价与控制价之间，按策略选取 |
| 价格分满分 | 来自招标文件评标办法 |
| 预估价格分 | 按招标文件公式试算 |

价格分公式与参数以当次招标文件为准，见 `knowledge_base/domain/procurement/references/rules/scoring_rules.md`。

---

## 3. 填写说明

- **人力**：按 WBS 或角色清单估算人天，单价参考历史项目或公司标准。
- **设备/软件**：与技术方案中的配置、License 一致，便于中标后履约。
- **利润率**：结合公司政策、竞争程度与价格分权重决定；价格分权重高时需更精细测算。
