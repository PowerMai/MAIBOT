# Digital Workforce Platform - 产品设计文档

## 一、产品愿景

**一句话**：为每个人提供可训练、可协作、可交易的数字员工，构建新型数字劳动力社会基础设施。

**人机关系**：人类是方法论导师（提供 Skills），数字员工是执行者（自动干活），人类参与决策。数字员工是结果导向的专家岗位，不是拆分的小工序。

**竞争力公式**：个体竞争力 = 拥有的数字员工数量 × 质量 × 协作效率

## 二、分阶段目标与卖点

### Phase 1：数字员工工作台（单机版）

- **目标**：让「解决方案专家」等角色能真正交付招投标等结果。
- **用户痛点**：招投标工作量大、专业门槛高、时间紧。
- **关键卖点**：上传招标文件，自动分析、生成方案、合规检查，比人快、成本趋近于零。
- **交付**：完善招投标 Skills + 领域知识库；知识库构建工作流（工具构建工具）；Electron 桌面打包与启动说明。

### Phase 2：数字员工增强站（联网版）

- **目标**：联网解锁更强能力，形成数据飞轮。
- **用户痛点**：本地模型能力有限，知识库静态，Skills 有限。
- **关键卖点**：云端专业知识库持续更新，高级模型按需使用，Skills 市场一键安装。

### Phase 3：数字劳务市场（平台版）

- **目标**：训练好的数字员工可接别人的任务赚钱。
- **用户痛点**：个人使用场景有限，他人需要能力但不会训练。
- **关键卖点**：像 Uber 一样「出租」数字员工；基于现有 TaskPanel + boardApi 开放组织/市场级看板。

### Phase 4：数字劳动力社会（生态版）

- **目标**：数字劳务公司、行业协会、标准体系。
- **延伸**：互联网数据 + 数字员工替代人类中介（房产、汽车、保险等）；企业 IT 系统 + 数字员工 = 自动化业务处理。

## 三、技术原则

- **不重复实现**：会话管理用 Thread + Checkpointer；长期记忆用 Store + langmem；Agent 编排用 StateGraph + SubAgent；人类决策用 Interrupt；流式用 SSE + streamMode；模型管理用 LangGraph configurable。
- **业务构建优先**：Phase 1 以内容建设（Skills + 知识库）为主，系统能力已就绪。
- **工具构建工具**：知识库构建工作流（knowledge-building Skill）让 Agent 能参与构建知识库与 Skills，支持更多人协作建设。

## 四、开源与商业化边界

- **开源**：Electron App 框架、LangGraph Agent 运行时、Skills 规范与基础 Skills、A2A 协议规范、基础数字员工模板、知识图谱引擎。
- **商业化**：专业知识库（持续更新）、高级 Skills 与工作流、云端模型 API、数字劳务市场平台、组织管理工具、质量评估与认证、知识蒸馏管道。
- **网络**：离线基础功能可用；联网渐进解锁；知识库增量缓存 + TTL，不做「锁」做「桥」。

## 五、与本系统实现的对应

- **单机版应用体系**：通用 Agent + 角色（Composer 选择）+ 对话框 + 编辑 + 资源管理 + 知识库 + 任务管理 + 通信管理 + 配置管理，已实现并联动。
- **角色**：即数字员工；由 `skill_profile`、知识域、提示词与质量标准定义；无需新建角色管理系统，丰富现有角色的 Skills 与知识库即可。
- **首阶段重点**：招投标「解决方案专家」可端到端交付投标方案；知识库构建工作流支持「文档→KG/本体→可选生成 Skill」；桌面运行与打包见 `frontend/desktop/DESKTOP.md`。

**规划落地与接续**：各 Phase 交付核对、未完成项及后续如何继续，见同目录下 `规划落地清单与后续接续.md`。
