# 数字员工角色矩阵（结果导向）

每个角色 = 一个为结果负责的专家，不是工序上的小工。数字员工 = 一组 Skills + 领域知识库 + 工作流 + 质量标准。

## 专业服务类

| 数字员工 | 输入 | 交付结果 | 核心 Skills | 知识库依赖 | 市场需求 |
|----------|------|----------|------------|-----------|----------|
| 解决方案专家 | 招标文件 | 完整投标方案 | bidding 全套 + solution-design | 招投标法规 + 行业方案 | 极高 |
| 合同顾问 | 合同文本 | 风险评估报告 + 修改建议 | contract-review + risk-assess | 法律法规 + 合同案例 | 极高 |
| 财务分析师 | 财务数据 | 分析报告 + 决策建议 | data-analysis + report-gen | 财务准则 + 行业基准 | 高 |
| 审计助理 | 业务数据 | 审计报告 + 合规清单 | compliance-verify + sampling | 审计准则 + 行业规范 | 高 |

## 本系统已实现角色（knowledge_base/roles/）

- **solution_expert**（解决方案专家）：skill_profile=bidding，招投标全流程，对应上表「解决方案专家」。
- **contract_specialist**（合同专员）：skill_profile=contract。
- **executive_assistant**（行政助理）：skill_profile=office。
- **data_analyst**（数据分析师）：skill_profile=analytics。
- **knowledge_manager**（知识管理）：skill_profile=knowledge，含知识库构建工作流。

角色通过 Composer 角色下拉选择；配置见各角色目录下 `config.json`。
