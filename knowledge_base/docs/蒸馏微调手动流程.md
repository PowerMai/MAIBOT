# 蒸馏微调手动流程

本文档用于补齐蒸馏闭环中的“手动微调/部署”环节，适配当前 MAIBOT 架构（本地模型为主，云端模型用于高质量样本生产与验证）。

## 1. 前置条件

- 已启用 `DistillationMiddleware`，并持续产生样本：
  - `knowledge_base/learned/distillation_samples.jsonl`
- 已有可执行基线模型（如 `qwen/qwen3.5-35b-a3b`）
- 可用的训练工具链（示例：Unsloth / Axolotl / LLaMA-Factory，三选一）

## 2. 导出训练集

使用统一入口脚本（会桥接到 legacy script）：

```bash
python backend/tools/upgrade/export_distillation_dataset.py \
  --input knowledge_base/learned/distillation_samples.jsonl \
  --output knowledge_base/learned/distillation_train.jsonl \
  --max 5000
```

说明：
- 输出格式为 `messages` 双轮结构（user/assistant）
- 建议保留最近高质量样本，避免混入旧低质分布

## 3. 质量筛选与 A/B 评估

在训练前先做评估，确认候选样本可用：

```bash
python backend/tools/upgrade/evaluate_distillation_ab.py
```

建议门槛：
- 关键任务（计划、代码、知识检索）通过率不低于当前线上基线
- 幻觉率、工具误用率不高于基线
- 若不达标，先清洗样本再训练

## 4. 微调执行（手动）

推荐流程：

1) 选择训练框架（Unsloth / Axolotl / LLaMA-Factory）
2) 载入 `knowledge_base/learned/distillation_train.jsonl`
3) 设置训练参数（LoRA/QLoRA 优先）
4) 训练后产出新模型权重（或适配器）
5) 本地推理冒烟测试（工具调用、长上下文、中文稳定性）

注意：
- 本项目不内置训练框架，避免重复造轮子
- 训练脚本建议放在独立目录（如 `backend/tools/training/`）并由 `python_run`/`shell_run` 调度

## 5. 注册新模型并灰度

### 5.1 模型注册

- 通过 API 或 `backend/config/models.json` 注册新模型
- 确保 `enabled=true`，并配置合理的 `context_length` 与 `config.max_tokens_*`

### 5.2 灰度验证

使用升级脚本进行策略化灰度：

```bash
python backend/tools/upgrade/auto_rollout_upgrade.py --stage canary
```

观察项：
- 成功率、平均响应时延、工具调用成功率
- 用户反馈与 `human_checkpoint` 信号
- 关键业务任务（如招投标链路）是否回归

通过后再逐步扩大流量（beta -> stable）。

## 6. 回滚策略

任一指标异常时执行：

1) 将默认模型切回上一版本
2) 保留本次评估与失败样本，写入 `knowledge_base/learned/`
3) 在 `EVOLUTION-SCORES.md` 记录回滚原因

## 7. 与系统能力对齐

- 采集：`DistillationMiddleware`（已集成）
- 导出：`backend/tools/upgrade/export_distillation_dataset.py`
- 评估：`backend/tools/upgrade/evaluate_distillation_ab.py`
- 灰度：`backend/tools/upgrade/auto_rollout_upgrade.py`
- 观测：LangSmith + 本地执行日志

以上构成可执行的“手动蒸馏闭环”：采集 -> 导出 -> 评估 -> 微调 -> 灰度 -> 回滚。
