# 数字员工技能体系

本系统作为数字员工可提供的**体系化技能**的归纳与设计说明。技能按 L1–L5 分层，通过 skill_profile 与角色映射按需加载，便于管理与 LLM 查找。

## 一、分层定义（L1–L5）

| 层级 | 含义 | 示例 |
|------|------|------|
| **L1 人的基本** | 通用推理、阅读、表达、验证 | reasoning、ask-methodology |
| **L2 工程师/工作方式基本** | 规划、调试、评审、进化 | plan、debug、review、self-learning、self-evolution、auto-discovery |
| **L3 办公文员基本** | 文档格式、表格、演示、产出 | pdf、docx、xlsx、pptx；市场：文档生成、报告生成、数据分析、合规、可视化 |
| **L4 角色专长** | 工具与知识工程 | skill-creator、mcp-builder、knowledge_engineering 下 7 个；市场：知识构建、本体管理、流程编排、外部资源 |
| **L5 专项** | 具体业务端到端技能 | **bidding**（招投标） |

- L1–L4：优先使用 Claude/本仓库/市场已有技能，不重复造轮子。
- L5：仅招投标一个专项，对外统一名称 `bidding`。

## 二、三源清单与层级归属

| 来源 | 位置 | 说明 |
|------|------|------|
| **内置** | `knowledge_base/skills/` | 22 个 SKILL（含 bidding）；按路径推断 domain/level |
| **插件** | `plugins/sales/skills/` | 招投标能力包（bidding-overview + 子技能）；经 skill_profile 注入路径后被发现 |
| **市场** | `backend/config/skills_market.json` | 12 条；安装/引用后纳入对应层级 |

内置技能按 L1–L5 的归属见《技能体系规划》中的归纳表。

## 三、角色 → 层级 → 专项映射

| 角色 | skill_profile | 加载层级 | 专项 |
|------|---------------|----------|------|
| 通用 Agent | general | L1+L2+L3+L4 全量 | 无 |
| 解决方案专家 | bidding | L1+L2+L3 + 招投标能力包 | **bidding（L5）** |
| 策略师 | bidding | 同解决方案专家 | bidding（L5） |
| 知识工程师 | knowledge_engineering | L1+L2+L3 + L4 知识工程 | 无 |
| 工程师/开发 | full | L1+L2+L4 | 无 |
| 分析师 | analyst | L1+L3+L4（数据/报告） | 无 |
| 办公助手 | office | L1+L3 | 无 |
| 合同专员 | contract | L1+L3 + 合规/合同 | 无 |

## 四、使用方式

- **发现**：`list_skills`、`match_skills`（按 profile/tier 裁剪）；`get_skill_info(name)`。match_skills 返回每条附带**匹配原因**（触发词/关键词/描述/语义），便于 UI 展示「推荐：bidding（与「招标文件」匹配）」及可观测性。
- **加载**：系统提示词内 skills_catalog 内联当前 profile 下的 name+短 description+何时使用；详细内容通过 `read_file(路径)` 或 `get_skill_info(name)` 读取 SKILL.md。
- **执行**：按 SKILL 内工作流调用工具（read_file、search_knowledge、python_run 等）；可选 `run_skill_script` 执行技能内脚本。
- **用户获得感**：对话中工具调用会展示「列出能力」「匹配能力」「查看技能详情」「执行技能脚本」；当 Agent 调用 get_skill_info 或 run_skill_script 时，前端显示「正在使用技能：xxx」，工具块标题旁会显示「技能：xxx」标签，用户可感知数字员工当前使用的技能。
- **来源与专业感**：catalog 与 list_skills/match_skills 中技能带来源标签（官方/内置/学习）；优先选用官方或有 scripts/ 的技能；交付前必须满足 SKILL 内「质量门」与「输出必含项」。前端技能列表与市场展示「官方」「已校验」标签。
- **配置**：`backend/config/skill_profiles.json` 的 paths 决定各 profile 加载范围；`data/skills_disabled.json` 可全局禁用某技能。单会话暴露技能数可由 `max_skills_per_session`（全局或 per-profile）限制，与 Claude「每请求技能数」思路一致，避免上下文过长；未配置或≤0 则不截断。
- **与 MCP 组合**：某角色下既有 skill_profile 决定的技能子集，也有已连接的 MCP；Agent 按技能说明中的流程与推荐工具（含 MCP 工具）执行，Skill 负责流程与标准，MCP 负责连接与数据。详见 [skills_claude_alignment.md](skills_claude_alignment.md) 中「Skills 与 MCP 分工」。

## 五、专业使用 vs 一般使用

- **一般使用（free）**：allow_skills 限 foundation/general/format/modes/community；max_custom_skills: 5。
- **专业使用（pro/enterprise）**：allow_skills: ["*"]，更高自定义技能上限；市场条目 `requires_tier` 控制安装与暴露。
- list_skills/BUNDLE 仅暴露**当前用户 tier 允许**的技能，详见《技能体系规划》与 `backend/config/license_tiers.json`。
