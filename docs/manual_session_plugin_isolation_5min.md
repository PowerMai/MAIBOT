# 会话级插件隔离人工补测（5 分钟）

目标：验证“会话插件开关”是线程级（thread-scoped），不会全局串线。

## 前置条件

- 已启动前后端开发服务。
- 至少存在两个线程（例如 A、B）。
- Composer 可见“插件(n)”按钮。

## 步骤

1. 在线程 A 打开插件面板。
2. 仅勾选一个易识别插件（建议 `sales`），记录显示为 `插件(1)`。
3. 在线程 A 发送一条消息：`请列出当前会话启用的插件并解释其能力`，确认输出包含 `sales`。
4. 切换到线程 B（确保左侧线程高亮已切换）。
5. 查看插件按钮计数：
   - 期望：不是 `1`（通常 `0` 或 B 自己历史值）。
6. 在线程 B 发送同样消息，确认输出不受线程 A 的 `sales` 开关影响。
7. 切回线程 A，检查插件按钮计数恢复为 `插件(1)`。
8. 在线程 A 取消勾选 `sales`，确认变为 `插件(0)`，再切 B，确认 B 状态不被 A 变化覆盖。

## 通过标准

- A/B 两线程的 `插件(n)` 可独立保持；
- A 线程启用插件后，B 线程不自动继承；
- 回切线程后状态可恢复；
- 每个线程行为与各自插件状态一致。

## 失败判定

- 任一线程切换后插件状态被全局覆盖；
- A 改动导致 B 同步变化；
- 线程回切后状态丢失或异常串线。
