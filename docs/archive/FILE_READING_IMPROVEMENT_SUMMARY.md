# 文件读取功能改进总结

## ✅ 已完成的改进

### 增强 `read_file` 工具，支持分页和分块读取

**文件**：`backend/tools/file_operations.py`

**改进内容**：
1. ✅ 支持**分块读取**：通过 `chunk_index` 参数读取不同分块
2. ✅ 支持**按行读取**：通过 `start_line` 和 `end_line` 参数读取指定行
3. ✅ 智能提示：自动提示如何继续读取剩余内容
4. ✅ 保持向后兼容：默认行为不变（读取前 4000 字符）

---

## 📋 使用方式

### 1. 默认读取（小文件）
```python
# 读取前 4000 字符（如果文件小于 4000 字符，读取完整文件）
content = await read_file("/files/123")
```

### 2. 分块读取（大文件）
```python
# 读取第 1 块（前 4000 字符）
content1 = await read_file("/files/123", chunk_index=0)

# 读取第 2 块（4000-8000 字符）
content2 = await read_file("/files/123", chunk_index=1)

# 读取第 3 块（8000-12000 字符）
content3 = await read_file("/files/123", chunk_index=2)
```

### 3. 按行读取（结构化文件）
```python
# 读取第 100-200 行
content = await read_file("/files/123", start_line=100, end_line=200)

# 读取从第 50 行到文件末尾
content = await read_file("/files/123", start_line=50)
```

### 4. 自定义分块大小
```python
# 使用 8000 字符的分块大小
content = await read_file("/files/123", chunk_size=8000, chunk_index=0)
```

---

## 🎯 解决的问题

### 问题 1：大文件无法完整读取
**之前**：❌ 4000 字符截断，无法继续读取
**现在**：✅ 支持分块读取，可以多次调用获取完整内容

### 问题 2：有限窗口无法显示完整内容
**之前**：❌ 只能看到前 4000 字符
**现在**：✅ 可以按需读取不同部分（分块或按行）

### 问题 3：无法精确定位内容
**之前**：❌ 只能从开头读取
**现在**：✅ 支持按行号精确定位

---

## 📊 性能对比

| 场景 | 之前 | 现在 |
|------|------|------|
| **小文件（< 4KB）** | ✅ 完整读取 | ✅ 完整读取（不变） |
| **中等文件（4KB-100KB）** | ❌ 只能读取前 4KB | ✅ 分块读取，可获取完整内容 |
| **大文件（> 100KB）** | ❌ 只能读取前 4KB | ✅ 分块读取，按需加载 |

## 💡 使用建议

### 对于 LLM（DeepAgent）

**场景 1：需要完整文件内容**
```
1. 先读取第 1 块：read_file("/files/123", chunk_index=0)
2. 如果提示还有内容，继续读取：read_file("/files/123", chunk_index=1)
3. 重复直到读取完整文件
```

**场景 2：只需要特定部分**
```
1. 如果是代码文件，按行读取：read_file("/files/123", start_line=100, end_line=200)
2. 如果是文本文件，分块读取：read_file("/files/123", chunk_index=2)
```

**场景 3：搜索特定内容**
```
1. 先读取第 1 块，查看文件结构
2. 根据内容判断需要读取哪些部分
3. 按需读取相关分块或行
```

## 📝 技术细节

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `file_path` | str | 必需 | 文件路径，格式：`/files/{id}` |
| `start_line` | int | 0 | 起始行号（从0开始） |
| `end_line` | int | None | 结束行号（可选） |
| `chunk_size` | int | 4000 | 分块大小（字符数） |
| `chunk_index` | int | 0 | 分块索引（从0开始） |

### 读取优先级

1. **按行读取**：如果指定了 `end_line` 或 `start_line > 0`
2. **分块读取**：如果指定了 `chunk_index > 0`
3. **默认读取**：读取前 `chunk_size` 个字符

### 错误处理

- ✅ 文件不存在：返回明确的错误信息
- ✅ 行号超出范围：返回错误信息和文件总行数
- ✅ 分块索引超出范围：返回错误信息和文件总长度
- ✅ 网络错误：返回详细的错误信息
