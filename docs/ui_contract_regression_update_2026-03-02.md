# UI/后端契约回归增量报告（2026-03-02）

## 本轮目标
- 收敛前后端在模式、角色、会话上下文、多窗口行为上的契约差异。
- 以最小改动优先修复高频用户路径上的一致性问题，避免功能回退。

## 已完成增量修复
- **模式契约收敛**
  - 前端 `debug/review` 不再互斥裁剪，改为都可见；`⌘4` 仅作为“动态第四模式”推荐入口。
  - `/plan`、`/debug`、`/review` 统一优先走后端 `/slash/execute`，前端仅保留兜底扩展。
- **角色-模式硬约束**
  - 后端在 `ModePermissionMiddleware` 增加角色可用模式校验；角色不允许的模式会在工具层被拒绝并返回结构化错误。
- **多窗口上下文隔离**
  - `maibot_workspace_path` 纳入窗口级存储（sessionStorage）策略，降低多窗口串扰。
  - 线程列表和线程检索链路改为读取窗口级工作区路径（保留历史全局键兜底迁移）。
- **任务与会话入口体验**
  - 删除非当前线程不再强制新建会话；固定线程行为与“置顶”文案保持一致。
- **通知与错误反馈**
  - 通知按工作区分桶存储，支持跨窗口 storage 同步。
  - 后端错误 toast 增加短时去重，降低重复提示噪音。

## 快速回归结论
- Ask/Plan/Agent/Review/Debug 的模式切换与后端执行语义已更一致。
- 角色模式展示、快捷键提示、后端权限判定三者已形成闭环。
- 多窗口下线程/工作区上下文串扰风险已下降（线程与工作区均窗口级优先）。

## 仍建议继续优化（P1）
- 将 `workspace_path` 的读取入口继续收敛到统一工具函数，减少直接 `localStorage` 访问点。
- 为“slash 统一通道”补一条自动化回归脚本（`/plan|/debug|/review` 三条最小用例）。
- 增补“角色-模式拒绝”前端可视化提示（当前已有错误信息，建议再增加引导文案）。
