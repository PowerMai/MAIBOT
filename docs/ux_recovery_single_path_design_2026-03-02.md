# 用户恢复单路径设计（P1）

## 目标

将“失败后恢复”收敛到单主入口，降低用户在 Thread / Dashboard / TaskDetail 三处来回切换的认知负担。

## 设计原则（对齐 Cursor/Cowork）

- 一个主入口：默认在 Thread 顶部展示“继续恢复”。
- 其他入口降级为二级动作（更多操作菜单）。
- 状态持久化以 thread 为粒度，避免全局覆盖。

## 迁移方案

1. 存储模型从全局键迁移为 `run_summary:{threadId}`。
2. Dashboard 只读取当前活动 thread 的恢复状态。
3. TaskDetail 通过事件总线跳转到 Thread 主入口，不再承载主恢复按钮。
4. 保留旧键 1 个版本周期做兼容读取，再移除。

## 验收指标

- 失败后到“可重试操作”的点击路径 <= 2 步。
- 跨 thread 切换时不出现错配恢复建议。
- 恢复成功率趋势上升（按周对比）。
