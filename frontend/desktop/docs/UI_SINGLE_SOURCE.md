# UI 单一信息源约定

界面上的**同一类信息或同一功能只在一处展示**，避免重复与歧义，从用户角度保持清晰一致。

## 已统一项

| 信息/功能 | 唯一展示位置 | 说明 |
|-----------|--------------|------|
| **后端连接状态** | 底部状态栏（左侧区域旁：绿点已连接 / 红点已断开） | 由 `checkHealth` 定期检查，Chat 顶栏不再显示 ConnectionStatusDot |
| **⌘K 命令面板** | 活动栏搜索图标 + 仪表盘快捷键提示 | 两处均为同一入口，文案统一为「命令面板 (⌘K)」 |
| **⌘J 打开 AI** | 活动栏 Sparkles 图标 + 仪表盘快捷键提示 | 文案统一为「AI 面板 (⌘J)」，不再使用「打开对话」 |
| **快捷键说明** | 仪表盘底部为简要提示；Composer 内 ⌘? 为完整快捷键列表 | 简要 vs 完整分工明确，不重复罗列 |

## 分工原则

- **底部状态栏**：应用级状态（未保存文件、当前文件/行列、连接状态、AI 运行中、AI 标识）。
- **活动栏**：导航与布局（文件、命令面板、知识库、任务、布局切换、AI 面板、设置）。
- **仪表盘**：入口页的快捷操作与最近对话，快捷键仅作提示，与活动栏 title 一致。
- **Chat 顶栏**：仅标题「AI」、对话标签列表、执行日志、关闭面板；不重复展示连接状态。

## 新增功能时注意

- 若新增「状态类」信息，优先放在底部状态栏或当前上下文最相关的一处，避免在多个面板重复显示。
- 若新增「入口」（如新快捷键），在活动栏或仪表盘中择一作为主入口，另一处至多做简短提示并保持文案一致。
