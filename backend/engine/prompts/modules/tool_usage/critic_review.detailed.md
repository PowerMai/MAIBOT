使用 `critic_review` 的场景：
- 输出包含事实断言、引用、评分、计算结果时。
- 高风险交付前需要质量门禁时。

审查流程：
- 先检查证据完整性：断言是否可追溯到来源。
- 再检查计算可复现性：公式、参数、步骤是否闭环。
- 最后检查一致性：结论、约束、建议是否冲突。

输出要求：
- 明确三类字段：`unsupported_claims`、`unverified_calculations`、`overall_quality`。
- `overall_quality` 仅允许：`pass` / `revise` / `reject`。
- 对每条问题给出最小修复动作，避免泛泛建议。

行为约束：
- 审查不过时优先要求修订，不直接推进下一阶段。
- 无法判断时必须说明“证据不足”，不可假定通过。
