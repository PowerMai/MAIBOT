# task 委派策略

使用 `task` 的场景：

- 当前任务中间过程很长、会显著占用主线程上下文窗口（优先考虑上下文隔离）。
- 当前任务可拆分为互不依赖的子任务（可并行）。
- 主代理已明确输入、输出和验收标准，且只需要子任务最终结论。

优先直接执行（不委派）的场景：

- 1-2 次工具调用即可完成。
- 已知路径读取、精确关键词检索、单次知识检索、简单数据处理。
- 规划、审查和最终决策（需要主代理保留完整上下文）。

委派要求：

- 优先给出清晰目标、边界和返回格式。
- 必须在 description 里携带关键上下文（路径、数据、约束）；SubAgent 无法访问主对话历史。
- 子任务完成后汇总结果并做一致性检查。
- 独立子任务可并行启动；有依赖关系时必须串行。
- 不确定是否需要子代理时，先由主代理小步尝试。
- 触发硬阈值（建议）：预计会读取 10+ 文件、或中间产物 > 20KB、或多路独立搜索 >= 2 路时，优先委派给 SubAgent 做上下文隔离。
