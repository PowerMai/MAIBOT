# Distillation Manual Workflow (Phase 2 - Manual)

This document defines the current manual workflow for distillation and rollout.
It is intentionally script-first to avoid duplicate pipeline code.

## Scope

- Export training/eval dataset from runtime traces
- Audit skill-hit quality
- Run A/B evaluation
- Draft rollout policy
- Execute rollout helper
- Review rollout runtime summary

## Prerequisites

- Backend dependencies installed
- Existing runtime logs / traces available
- Run from project root: `ccb-v0.378`

## 1) Export Distillation Dataset

```bash
python3 backend/tools/upgrade/export_distillation_dataset.py
```

Expected:
- Exit code `0`
- Dataset export artifact generated by legacy auto-discovery script

## 2) Audit Distillation Skill Hits

```bash
python3 backend/tools/upgrade/distillation_skill_hit_audit.py
```

Expected:
- Exit code `0`
- Skill hit diagnostics available for review

## 3) Evaluate Distillation A/B

```bash
python3 backend/tools/upgrade/evaluate_distillation_ab.py
```

Expected:
- Exit code `0`
- A/B comparison output generated

## 4) Generate Rollout Policy Draft

```bash
python3 backend/tools/upgrade/generate_rollout_policy_draft.py
```

Expected:
- Exit code `0`
- Policy draft file generated

## 5) Execute Auto Rollout Helper

```bash
python3 backend/tools/upgrade/auto_rollout_upgrade.py
```

Expected:
- Exit code `0` (or explicit non-zero with error reason)
- Bridge logs written to `data/upgrade_bridge_log.jsonl`

## 6) Review Runtime Rollout Summary

```bash
python3 backend/tools/upgrade/rollout_runtime_summary.py
```

Expected:
- Exit code `0`
- Runtime summary output available for operator review

## Error Handling

All wrappers use the legacy bridge:
- Source: `backend/tools/upgrade/_legacy_bridge.py`
- On failure, bridge prints a compact reason to stderr
- Detailed tails are appended to:
  - `data/upgrade_bridge_log.jsonl`
  - keys include `exit_code`, `stderr_tail`, `stdout_tail`

## Notes

- Current phase is manual by design (operator-supervised).
- Next phase can automate orchestration after metrics and policy are stable.
